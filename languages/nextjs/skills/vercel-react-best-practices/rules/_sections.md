# セクション定義

このファイルは全セクションの並び順・影響度・説明を定義する。括弧内の
セクション ID は、ルールをグルーピングするためのファイル名接頭辞でもある。

---

## 1. ウォーターフォール解消 (async)

**影響度:** CRITICAL
**説明:** ウォーターフォールはパフォーマンスを最も損なう要因。逐次的な await のたびにネットワーク遅延が積み重なる。ここを潰すと最大の改善が得られる。

## 2. バンドルサイズ最適化 (bundle)

**影響度:** CRITICAL
**説明:** 初期バンドルサイズを減らすと、Time to Interactive と Largest Contentful Paint が改善する。

## 3. サーバサイドパフォーマンス (server)

**影響度:** HIGH
**説明:** サーバサイドレンダリングとデータ取得を最適化することで、サーバ側のウォーターフォールを解消し、応答時間を短縮できる。

## 4. クライアントサイドデータ取得 (client)

**影響度:** MEDIUM-HIGH
**説明:** 自動重複排除と効率的なデータ取得パターンによって、冗長なネットワークリクエストを減らせる。

## 5. 再レンダリング最適化 (rerender)

**影響度:** MEDIUM
**説明:** 不要な再レンダリングを減らすことで、無駄な計算を最小化し、UI の応答性を高められる。

## 6. レンダリングパフォーマンス (rendering)

**影響度:** MEDIUM
**説明:** レンダリング処理を最適化することで、ブラウザがやるべき仕事を減らせる。

## 7. JavaScript パフォーマンス (js)

**影響度:** LOW-MEDIUM
**説明:** ホットパスでのマイクロ最適化は積み重ねると意味のある改善になる。

## 8. 上級パターン (advanced)

**影響度:** LOW
**説明:** 慎重な実装が必要な特定ケース向けの上級パターン。
